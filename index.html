Why This Topic Is Interesting?


The result of an upcoming NBA match is always a popular topic among NBA basketball fans. People, especially NBA fans, always have a drastic debate on the results of games. They may support their hometown team, or support the team for which their favourite player plays for.

In addition, NBA game results, especially the score differences, are often exploited by lottery companies, which means these predictions could have business value. As long as we have a well-performed predictor, we can not only foresee the result of a single game, but also apply it so that people can make money from lotteries.

In this project, our target is to predict the score difference between two arbitrary teams. However, we won’t provide the difference as a precise number, instead, a difference range the difference will be in is rendered.


Data collections

We collect data from BASKETBALL-REFERENCE, a professional basketball database. To predict sports game results, data amount is considered very important so that we pick all game records over the last four NBA regular seasons – 5000 records in all. In addition, we are provided more than 20 attributes for each game record by this website, which helps us a lot in creating data models.

FEATURE SELECTION & DATA PROCESSING

Feature selection and data processing are two key parts of our data model as it directly affects our model accuracy. In this project, 27 features are chosen manually to be our data attributes and several data processing techniques are employed to optimize our model.

FEATURE SELECTION

Specifically, we manually choose FG, FGA, FG%, 2P, 2PA, 2P%, 3P, 3PA, 3P%, FT, FTA, FT%, PTS, ORtg, FTr, 3PAr, TS%, eFG%, FT/FGA, ORB, DRB, TRB, AST, STL, BLK, TOV and PF these 27 features as our data attributes.

DATA PROCESSING

Data tuple format

Data processing is one of the parts of our project that contains most workload. The reasons for that we cannot simply use the game records directly provided by online database are:

  • We need to generate a small database for each single game record.

  • We need to give each past games different affect weight according to how far away a game is before the target game.

  • We try to unify the attribute difference scale in number.
